Good parts

  • Loose typing
  • Functions as first class objects

  • Expresive object literal

Loose typing

  • Primitives: Number, Boolean, String.
  • Objects: Object, Function, Array, Date, RegExp, etc.
  • Special cases: Null, Undefined.

Loose typing



var a = 13; // Number declaration
var b = "thirteen"; // String declaration

  • Loose typing means that variables are declared without a type.
  • Variables in JavaScript are typed, but that type is determined internally.

Functions as first class objects

  • A function is an instance of the Object type
  • A function can have properties and has a link back to its constructor method
  • You can store the function in a variable
  • You can pass the function as a parameter to another function
  • You can return the function from a function

Functions as first class objects


var myBarFunc = function test(foo) {
  foo += 1;
  return foo;
};

var getKeys = function(obj){
Object.getOwnPropertyNames(obj).forEach(function(val, idx, array) {
  console.log(val + " -> " + obj[val]);
});
};

getKeys(myBarFunc);

consoles:

"prototype -> [object Object]"
"length -> 1"
"name -> test"

Expresive object literal

  • JSON like object literal

Dynamic objects

  • You can fully modify objects at runtime

var person = {
  firstName: "John",
  lastName: "Doe",
  gunType: "Revolver",
  sayHi: function() {
    console.log("Howdy");
  }
};

console.log(person.firstName + " " + person.lastName); //GET
person.sayHi(); //GET
person.gunType = "Shotgun"; //SET = update key
person["shoe-type"] = "Cowboy Boots"; //Other Notation
person.fireGun = function() {
  console.log("BAM");
}; //SET = create key

person.fireGun(); //GET
delete person.fireGun; //REMOVE

Function scope

  • Scope controls the visibility and lifetimes of variables and parameters.
  • Reduces naming collisions and provides automatic memory management.
  • Can someone see the error?

var foo = function() {
  var a = 3, b = 5;

  var bar = function() {
    var b = 7, c = 11;
    // At this point, a is 3, b is 7 and c is 11

    a += b + c;
    // At this point, a is 21, b is 7 and c is 11
  };
  // At this point, a is 3, b is 7 and c is undefined

  bar();
  // At this point, a is 21, b is 5
}

Closure

  • Whenever you see the function keyword within another function, the inner function has access to variables in the outer function context.
  • You can understand this best by example.

function foo(x) {
  var tmp = 3;
  return function (y) {
    alert(x + y + (++tmp)); // will also alert 16
  }
}

var bar = foo(2); // bar is now a closure.
bar(10);
/////////// another example ///////////

var quo = function(status) {
  return { // return an object
      get_status: function() {
      return status; // access status from outer function
    }
  }
}

var myQuo = quo("amazed");
console.log(myQuo.get_status()); // outputs amazed

typeof

      
// Numbers
typeof 37 === 'number';

// Strings
typeof "" === 'string';
typeof "bla" === 'string';
typeof (typeof 1) === 'string'; // typeof always return a string
typeof String("abc") === 'string'; // but never use this form!


// Booleans
typeof true === 'boolean';
typeof false === 'boolean';
typeof Boolean(true) === 'boolean'; // but never use this form!


// Undefined
typeof undefined === 'undefined';
typeof blabla === 'undefined'; // an undefined variable


// Objects
typeof {a:1} === 'object';

// use Array.isArray or Object.prototype.toString.call
// to differentiate regular objects from arrays
typeof [1, 2, 4] === 'object';

typeof new Date() === 'object';


// The following is confusing. Don't use!
typeof new Boolean(true) === 'object';
typeof new Number(1) === 'object';
typeof new String("abc") === 'object';


// Functions
typeof function(){} === 'function';
typeof Math.sin === 'function';

Global variables

  • A global variable is a variable that is visible in every scope.
  • Use of global variables degrades the reliability of the programms that use them.
  • 3 ways of defining globals
  • examples: Drupal, $, _, etc
      
var foo = value; // outside of any scope
window.foo = value; // anywhere in the code
foo = value; // anywhere in the code IMPLIED GLOBAL

+ Adds and concatenates

  • In JAVA this isn't an issue, in javascript however the compiler doesn't throw type errors.
  • Example:
      
var a = 10;
var b = 20;
console.log("result is " + a + b);
// RESULT IS 1020

// fast solution
console.log("result is " + (a + b));
// RESULT IS 30

parseInt

  • String function thst converts a String into Integer
  • Stops when it sees a non digit, "16" and "16 tons" would produce same result
  • If the input string begins with "0", radix is eight (octal) or 10 (decimal). Exactly which radix is chosen is implementation-dependent. ECMAScript 5 specifies that 10 (decimal) is used, but not all browsers support this yet. For this reason always specify a radix when using parseInt.

Phony arrays

  • Javascript doesn't have arrays, it has hashtables
  • slower, but no index out of bounds errors.
  • Harder to check for array existence.
      
var myArray = [1, 2, 3];

//WRONG WAY
if(typeof myArray === 'array') {
  // NOT ENTERED
}

//RIGHT WAY
if(Object.prototype.toString.apply(myArray) === "[object Array]") {
  // myArray is truly an array
}

JSHint

  • JSHint is a program that flags suspicious usage in programs written in JavaScript.
  • It is a community driven npm module, unlike JSLint by Douglas Crockford
  • It detects the bad parts.
  • Grunt integration.
  • configuration levels and code editor support

Instalation

npm install jshint -g



Configuration

  • via the --config flag from the command line
  • In package.json under jshintConfig key
  • In a .jshintrc file in your project folder
