What is vibe coding?

Vibe coding is a new way of coding where you just put in one prompt and then you can rest a bit in your chair and let AI do the work for you. Go grab a coffee, go chat with one of your beloved colleagues about your fabulous dreams of quitting your job and opening a beach bar that vanish every time payroll hits. [PAUSE]

This Sounds great, right? Well, it is. But it's not. [SPACE]

Vibe coding is a programming paradigm dependent on artificial intelligence where developers describe what they want in natural language and let AI generate the code for them.

The term was introduced and coined by Andrej Karpathy in February 2025 and was added to the Merriam-Webster Dictionary the following month as a "slang & trending" noun.

Some cool projects have already been made. And A good percentage of global code is now already ai generated, weather that is a good thing or not is what we'll discover along the way.

How can I use this new methodology?

Back to that 2nd question. How can I use this new methodology?

Well there are a couple of things you need to know before you begin.

The first thing you need to realize is that Coding and Programming are not the same.[SPACE]

Coding != Programming

Coding is the skill of translating logic into instructions for a computer by writing code. An elite coder might be able to bang out a hundred words per minute of perfect C code in VIM purely from memorized syntax, but large language models are making this skill obsolete. An eight year old prompter can now write code faster than a skilled professional with twenty years of experience.

Programming on the other hand is a far broader skill that's kinda hard to describe. It's both an art and a science that requires ingenuity, creativity, and good instincts gained throughout years of experience.

The best programmers often spend more time removing code than writing code. And if you don't understand the code you're vibing out like a true programmer, you are in for a really wild ride.

I'm pretty sure that some of you who use ai for coding have already experienced this kind of flow where you ask AI to fix your code and it actually gets worse and worse. So how do we avoid this? How do we set ourselves up for success.

Are there any rules I can apply?

Are there any rules you can apply? Yes there are, there are even some techniques like the CREATE technique on how to write good prompts. Remember that one?

  • C for Contextualise or set up a Character,
  • R for the Request,
  • E for Elaboration or showing Examples,
  • A for Include Authenticity or small Adjustments,
  • T to Make some Transformation or change the Type of the output
  • and E for some extras.

First lets define some rules, then we'll go a little bit deeper with the techniques.

Rules

  1. Choose the right stack
  2. Get gud at Git
  3. Think like a PM

The first rule and this rule is the most flexible one is to choose the right stack you are building in. And what is considered to be the RIGHT stack? Well, for starters your new fancy javascript framework is probably going to kill your vibez because ai might not have had enough training data to get good at it.

You wanna choose something popular that already exists for a while like: React, Svelte or tailwind, express, node, fastApi you name it.

In terms of languages choose one of the popular ones, and if you are a coder and already know some languages it certainly at your advantage to choose something you already know. (duh)

[SPACE] The second rule is...Get good at git. Some people in the crowd know me as a git nerd because I know all the commands like a bit of a freak, but hey I like the terminal.

Git is a version control system that allows you to keep track of changes to your code and collaborate with others. It's essential for any software development project.

When something goes wrong while vibe coding...you can always fall back to your latest commit or latest stashed changes.[SPACE]

The third rule is to think like a PM. Maybe for the devs among us a harder step. But again this depends on your personal skills.

Ask not what can AI do for me? but what can I do for AI?

You are not a coder anymore...

You are now a product manager.

And if you wanna be a good one, you need to break down complex requirements into small steps. They should be highly specific and also provide the proper context to solve the problem.

You want to give it things like recent documentation and images for UI design. It might sound like a vibe killer, but you don't want your LLM to get too creative. You want it to be as deterministic as possible.

V.I.B.E.S.

  • Verify continuously
  • Iterate in inches
  • Blueprint before you build
  • Explain and understand
  • Scope your prompts
  • Keep your vibe coding sessions on vibes.
  • [SPACE] Verify continuously - Test & Review AI output every step of the way. Never ship code that you haven't run and inspected.
  • [SPACE] Iterate in inches - Prompt -> run -> refine -> repeat To steer the model and catch mistakes early on.
  • [SPACE] Blueprint before you build - this is the most important step of the workflow. This is the same in regular coding. We often first go to the drawing board and choose how we're going to build what we're about to build.
  • You really want to spend a lot of time on this step. Wou want a detailed, thorough plan that you can pass to your agent so it knows exactly what you want.
