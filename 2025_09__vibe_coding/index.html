What is vibe coding?

Vibe coding is a new way of coding where you just put in one prompt and then you can rest a bit in your chair and let the AI do the work for you. Sounds great, right? Well, it is. But it's not.

Vibe coding is a programming paradigm dependent on artificial intelligence where developers describe what they want in natural language and let AI generate the code for them. The term was introduced and coined by Andrej Karpathy in February 2025 and was added to the Merriam-Webster Dictionary the following month as a "slang & trending" noun.

How can I use this new methodology?

Back to that question. How can I use this new methodology?

Well there are a couple of things you need to know before you begin

The first thing you need to realize is that Coding and Programming are not the same.

Coding != Programming

Coding is the skill of translating logic into instructions for a computer by writing code. An elite coder might be able to bang out a hundred words per minute of perfect C code and VIM purely from memorized syntax, but large language models are making this skill obsolete. An eight year old prompter can now write code faster than a skilled professional with twenty years of experience.

But programming is a far broader skill that's kinda hard to describe. It's both an art and a science that requires ingenuity, creativity, and good instincts gained throughout years of experience.

So something to bear in mind. And that comes back to that perspective of which we already looked at.

Are there any rules I can apply?

Are there any rules you can apply? Yes there are, there are even some techniques like the CREATE technique we often use to write good prompts. Remember that one?

C for Contextualise or set up a Character, R for the Request, E for Elaboration or showing Examples, A for Include Authenticity or small Adjustments, To to Make some Transformation or change the Type of the output and E for some extras.

First lets define some rules and then we'll go a little bit deeper with the techniques.

Rules

  1. Choose the right stack
  2. Get gud at Git
  3. Think like a PM

The first rule is to choose the right stack you are building in. And what is considered to be the RIGHT stack? Well, for starters your new fancy javascript framework is probably going to kill your vibez because ai might not have had enough training data to get good at it.

You wanna choose something popular that already exists for a while like: React, Svelte or tailwind, express, node, fastApi you name it.

In terms of languages choose one of the popular ones, and if you are a coder and already know some languages it certainly at your advantage to choose something you already know. (duh)

[SPACE] The second rule is...Get good at git. Some people in the crowd know me as a git freak because I know all the commands like a bit of a freak, but hey I like the terminal. Git is a version control system that allows you to keep track of changes to your code and collaborate with others. It's essential for any software development project.

When something goes wrong while vibe coding...you can always fall back to your latest commit or latest stashed changes.

The third rule is to think like a PM. Maybe for the devs among us a harder step. But again this depends on your personal skills.

Ask not what can AI do for me? but what can I do for AI?
