  • Hoe kleiner het scherm hoe minder we zien boven de fold, en dus als gevolg hoe interessanter het scrollmechanisme wordt.

How much do we scroll?

Let's do the math.

90 minutes/day * 30cm/5sec = 324 m/day

324 m/day * 365d = 118km

  • Als we gemiddeld 90 min/dag op onze smartphone of laptop spenderen.
  • En op dat toestel 30cm per 5sec scrollen.
  • Dan scrollen we ongeveer 324 meter per dag.
  • Als we 324 meter per dag scrollen dan scrollen we ongeveer 118km per jaar.

Common tools for scrolling

Box Overflow Control

.selector {
  overflow: auto | visible | hidden | scroll;
  overflow-x: auto | visible | hidden | scroll;
  overflow-y: auto | visible | hidden | scroll;
}
  • Ik denk 1 van de meest bekende css properties als het gaat over scrolling is de overflow property uit de css level 2 spec. We gebruiken deze wanneer de content te groot is voor de bestemming. Overflow x en y kwamen later in de overflow module 3 spec.
  • Default op auto, wil zeggen als de content te groot wordt dan verschijnt de scrollbar.
  • Er zijn 2 types overflow: ink overlow en scrollable overflow. Ink overlow zijn elementen zoals box shadow, border-images, text-decoration, etc
    Scrollable overflow is de content dat uit de de content box valt.

Text Overflow Control

.selector {
  text-overflow: clip | ellipsis;
  white-space: normal | nowrap | pre | pre-wrap | pre-line | break-spaces;
}
  • Met text-overflow kunnen we drie puntjes acter en text zetten wanneer die te lang wordt. Een voorwaarde hierbij is wel dat de container de css => overflow: hidden; en white-space: nowrap; heeft.
  • Er is nog een probleem met text overflow, de text overflow property heeft enkel invloed op de inline progressie richting. dus deze property kan niet werken over verschillende lijnen.
  • normal: lijnen worden onderbroken wanneer nodig om de lineboxes te vullen.
    nowrap: lijnen worden niet onderbroken. Dus geen linebreaks.
    pre: sequenties van whitespaces wordt behouden. Lijnen worden enkel onderbroken op brs en op newline karakters.
    pre-wrap: sequenties van whitespaces wordt behouden. Lijnen worden onderbroken op brs en op newline karakters maar ook om de line box te vullen.
    pre-line: sequenties van whitespaces wordt toegeklapt. Lijnen worden onderbroken op brs en op newline karakters en ook op lijnen om de line box te vullen.
    break-spaces: is zoals pre wrap maar dan met uitzodering van sequenties van whitespaces en tabs wordt behouden ook op het einde van de lijn.
